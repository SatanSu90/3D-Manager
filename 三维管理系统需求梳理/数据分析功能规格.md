# 数据预览增强 + 数据分析与指标管理功能规格

## 一、功能概述

### 功能一：数据预览增强
在现有数据源管理基础上，增强数据预览能力：字段类型展示、数据概览统计、预览结果导出。

### 功能二：数据分析引擎
支持单/多数据源数据分析，提供数据清洗、过滤、基础统计、高级统计、机器学习方法，分析结果保存为指标供可视化图表和三维模型使用。

## 二、数据库设计

### 2.1 新增表

```sql
-- 分析任务表
CREATE TABLE analysis_task (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    status ENUM('DRAFT','RUNNING','SUCCESS','FAILED') NOT NULL DEFAULT 'DRAFT',
    owner_id BIGINT NOT NULL,
    config TEXT COMMENT 'JSON: 任务全局配置(多数据源关联配置)',
    result_summary TEXT COMMENT 'JSON: 执行结果摘要',
    error_message TEXT,
    executed_at DATETIME,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_task_owner (owner_id),
    INDEX idx_task_status (status),
    CONSTRAINT fk_task_owner FOREIGN KEY (owner_id) REFERENCES user(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 分析步骤表（一个任务包含多个步骤，按 step_order 顺序执行）
CREATE TABLE analysis_step (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    task_id BIGINT NOT NULL,
    step_order INT NOT NULL,
    step_type ENUM('CLEAN','FILTER','STATS','ADVANCED_STATS','ML') NOT NULL,
    step_name VARCHAR(200) NOT NULL,
    config TEXT NOT NULL COMMENT 'JSON: 步骤配置参数',
    result TEXT COMMENT 'JSON: 步骤执行结果',
    status ENUM('PENDING','SUCCESS','FAILED') NOT NULL DEFAULT 'PENDING',
    INDEX idx_step_task (task_id),
    CONSTRAINT fk_step_task FOREIGN KEY (task_id) REFERENCES analysis_task(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 分析数据源关联表（一个任务可关联多个数据源）
CREATE TABLE analysis_data_source (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    task_id BIGINT NOT NULL,
    data_source_id BIGINT NOT NULL,
    alias VARCHAR(100) NOT NULL COMMENT '数据源别名，用于SQL中引用',
    table_name VARCHAR(200) COMMENT '选定的表名',
    query_sql TEXT COMMENT '自定义SQL（可选，优先于table_name）',
    INDEX idx_ads_task (task_id),
    CONSTRAINT fk_ads_task FOREIGN KEY (task_id) REFERENCES analysis_task(id) ON DELETE CASCADE,
    CONSTRAINT fk_ads_ds FOREIGN KEY (data_source_id) REFERENCES data_source(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 指标表（分析结果保存为可复用指标）
CREATE TABLE indicator (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    type ENUM('ATOMIC','DERIVED','COMPOSITE') NOT NULL DEFAULT 'ATOMIC',
    value_type ENUM('NUMBER','STRING','JSON','TABLE') NOT NULL DEFAULT 'NUMBER',
    value TEXT COMMENT '指标值(JSON)',
    data_source_id BIGINT COMMENT '来源数据源',
    task_id BIGINT COMMENT '来源分析任务',
    owner_id BIGINT NOT NULL,
    tags VARCHAR(500) COMMENT '标签(逗号分隔)',
    visibility ENUM('PRIVATE','DEPARTMENT_SHARED','PUBLIC') NOT NULL DEFAULT 'PRIVATE',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_indicator_owner (owner_id),
    INDEX idx_indicator_type (type),
    CONSTRAINT fk_indicator_owner FOREIGN KEY (owner_id) REFERENCES user(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 3.3 DataAnalysisService — 核心分析引擎

```java
@Service
public class DataAnalysisService {
    // 任务管理
    - createTask(request, username)
    - getTask(id)
    - listTasks(keyword, ownerId, page, size)
    - deleteTask(id)
    
    // 步骤管理
    - addStep(taskId, request)
    - updateStep(stepId, request)
    - deleteStep(stepId)
    - reorderSteps(taskId, stepOrders)
    
    // 执行引擎
    - executeTask(taskId)  // 按顺序执行所有步骤
    - executeStep(step, inputData)  // 执行单个步骤
    
    // 分析方法实现
    // L1 数据清洗
    - cleanData(inputData, config)  // config: {removeNull: true, dedup: true, typeConversion: {...}, outlierRemoval: {field, method}}
    // L2 数据过滤
    - filterData(inputData, config)  // config: {conditions: [{field, operator, value}], selectFields: [], orderBy: {field, direction}, limit: N}
    // L3 基础统计
    - basicStats(inputData, config)  // config: {fields: [], methods: [MEAN, MEDIAN, STD_DEV, MIN, MAX, COUNT, PERCENTILE]}
    // L4 高级统计
    - advancedStats(inputData, config)  // config: {type: CORRELATION|GROUP_BY|CROSS_TAB, params: {...}}
    // L5 机器学习
    - mlAnalysis(inputData, config)  // config: {algorithm: LINEAR_REGRESSION|KMEANS|DECISION_TREE, params: {...}}
    
    // 指标保存
    - saveAsIndicator(taskId, request)  // 将分析结果保存为指标
}
```

### 3.4 分析方法详细设计

#### L1 数据清洗 (CLEAN)
```json
{
  "removeNull": true,
  "nullStrategy": "DROP_ROW",  // DROP_ROW | FILL_DEFAULT
  "fillValues": {"field1": 0, "field2": "unknown"},
  "dedup": true,
  "dedupFields": ["field1", "field2"],
  "typeConversion": {"field1": "INTEGER", "field2": "DOUBLE"},
  "outlierRemoval": {
    "field": "value",
    "method": "IQR",  // IQR | Z_SCORE
    "threshold": 1.5
  }
}
```

#### L2 数据过滤 (FILTER)
```json
{
  "conditions": [
    {"field": "age", "operator": "GTE", "value": 18},
    {"field": "status", "operator": "EQ", "value": "active"}
  ],
  "logic": "AND",
  "selectFields": ["name", "age", "status"],
  "orderBy": {"field": "age", "direction": "DESC"},
  "limit": 100
}
```
运算符: EQ, NEQ, GT, GTE, LT, LTE, LIKE, IN, BETWEEN, IS_NULL, NOT_NULL

#### L3 基础统计 (STATS)
```json
{
  "fields": ["salary", "age"],
  "methods": ["COUNT", "MEAN", "MEDIAN", "STD_DEV", "MIN", "MAX", "Q1", "Q3", "SUM"]
}
```
输出: 每个字段的统计结果表

#### L4 高级统计 (ADVANCED_STATS)
```json
// 相关性分析
{"type": "CORRELATION", "fields": ["salary", "age", "experience"]}
// 分组聚合
{"type": "GROUP_BY", "groupBy": ["department"], "aggregations": [{"field": "salary", "function": "AVG"}, {"field": "id", "function": "COUNT"}]}
// 交叉分析
{"type": "CROSS_TAB", "rowField": "department", "colField": "gender", "valueField": "salary", "function": "AVG"}
```

#### L5 机器学习 (ML)
```json
// 线性回归
{"algorithm": "LINEAR_REGRESSION", "features": ["age", "experience"], "target": "salary"}
// K-Means聚类
{"algorithm": "KMEANS", "fields": ["salary", "age"], "k": 3, "maxIterations": 100}
// 决策树分类
{"algorithm": "DECISION_TREE", "features": ["age", "salary", "experience"], "target": "department"}
```
